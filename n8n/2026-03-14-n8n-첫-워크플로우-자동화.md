### Context
n8n을 활용하여 첫 번째 워크플로우 자동화를 구축했습니다. 이 워크플로우는 디스코드(Discord)에서 특정 메시지가 수신되는 것을 트리거로 하여 일련의 자동화된 작업을 수행합니다. 주요 목표는 수신된 메시지를 바탕으로 TIL(Today I Learned) 형식의 기술 문서를 생성하고, 이를 GitHub 저장소에 푸시하며, 최종적으로 README.md 파일에 해당 문서의 인덱스를 추가한 후, 작업 완료 알림 메시지를 발송하는 것입니다.

### Core
n8n 워크플로우는 다음과 같은 핵심 단계로 구성될 수 있습니다. 각 단계는 n8n 노드를 통해 구현되며, 노드 간의 데이터 흐름(Input/Output)을 이해하는 것이 중요합니다.

*   **Discord Trigger**: 특정 디스코드 채널에서 메시지 수신을 감지하여 워크플로우를 시작합니다.
    *   메시지 내용, 발신자 정보 등을 다음 노드로 전달합니다.
*   **AI Agent (또는 Custom Code) 노드**: 수신된 디스코드 메시지를 분석하고, 이를 바탕으로 TIL 형식에 맞는 마크다운(Markdown) 문서를 생성합니다.
    *   메시지 내용을 입력으로 받아, 정해진 TIL 구조(Context, Core, Insight)에 맞춰 문서를 구성하는 로직을 수행합니다.
*   **GitHub 노드 (파일 생성/업데이트)**: 생성된 TIL 마크다운 파일을 GitHub 저장소의 특정 경로에 푸시합니다.
    *   파일 이름, 내용, 커밋 메시지 등을 입력으로 받아 GitHub API를 통해 파일을 생성하거나 업데이트합니다.
*   **GitHub 노드 (README.md 업데이트)**: 새로 생성된 TIL 문서의 링크를 저장소의 `README.md` 파일에 추가하여 인덱싱합니다.
    *   `README.md` 파일의 현재 내용을 가져와 새 TIL 문서의 링크를 추가한 후, 업데이트된 내용을 다시 푸시합니다.
*   **Discord 노드 (완료 알림)**: 모든 자동화 과정이 성공적으로 완료되면, 작업 성공 메시지를 디스코드 채널로 발신합니다.
    *   성공 여부, TIL 문서 링크 등 필요한 정보를 입력으로 받아 디스코드 메시지를 발송합니다.

### Insight
n8n을 활용한 워크플로우를 처음 구축하면서, 노드의 '입력(Input)'과 '출력(Output)' 데이터 구조를 정확히 이해하는 것이 워크플로우 설계의 핵심임을 깨달았습니다. 각 노드가 어떤 형태의 데이터를 받아들이고, 어떤 형태의 데이터를 다음 노드로 전달하는지에 대한 명확한 이해는 복잡한 자동화 흐름을 효율적으로 구성하는 데 필수적입니다. 이러한 이해를 바탕으로 각 노드의 역할을 정의하고 데이터를 적절히 변환하면, 복잡해 보이는 자동화도 체계적으로 구현할 수 있습니다.
